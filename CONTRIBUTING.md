# Contributing to TerminalOS

Thank you for your interest in contributing to TerminalOS.

## Development Setup

1. Install Rust stable (1.85+)
2. Clone the repository
3. Build the workspace: `cargo build`
4. Run tests: `cargo test --workspace`
5. Run clippy: `cargo clippy --workspace --all-targets -- -D warnings`

## Code Standards

- **No placeholders** — every feature must compile and work
- **No TODO comments** — implement or open an issue
- **Zero warnings** — `cargo clippy` must pass with `-D warnings`
- **Tests** — add tests for new logic where meaningful
- **Formatting** — run `cargo fmt` before submitting
- **No AI attribution** — do not add `Co-authored-by: Cursor`, tool co-author trailers, or "generated by AI" disclaimers to commits, code, or docs

### Commit Message Policy

Commits must reflect human authorship only. Before your first commit, install the message hook:

```bash
chmod +x scripts/hooks/commit-msg
ln -sf ../../scripts/hooks/commit-msg .git/hooks/commit-msg
```

This automatically strips any accidental Cursor or AI co-author lines from commit messages.

## Pull Request Process

1. Fork the repository
2. Create a feature branch from `main`
3. Make focused, incremental changes
4. Ensure CI passes (build, test, clippy, fmt)
5. Write a clear PR description with test plan

## Architecture Guidelines

- Use traits for abstraction between crates
- No global state — dependency injection via `AppContext`
- Async for I/O-bound operations (Tokio)
- Configuration-driven behavior via TOML
- Security: never auto-execute AI-generated commands

## Reporting Issues

Include:
- OS and Rust version
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs or screenshots
